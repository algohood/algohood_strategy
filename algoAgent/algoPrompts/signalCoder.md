# 方案说明
## 方案分为九个部分：
- 信号名称: 作为代码脚本名称
- 标的数量: 作为数据结构中标的数量，若为单标的信号，则只对**第一个标的**进行计算，而不需要遍历所有标的
- 版本: 作为版本号
- 信号概述: 写入类注释
- 设计模式: 写入类注释
- 特征提取: 写入特征提取函数注释
- 信号计算: 写入信号计算函数注释
- 方向判断: 写入方向判断函数注释
- 参数说明: 写入参数说明
- 使用说明: 写入类注释

## 方案设计模式说明
- 方案基于对状态管理得复杂度，分为三种模式，每种模式适用于不同的交易场景和计算要求。
  * 无状态模式: 整个信号计算仅依赖当前批次数据（无需维护历史状态）
  * 半无状态模式: 整个信号计算不仅依赖当前批次数据，还需要依赖少量状态变量（需要维护历史状态）
  * 有状态模式: 整个信号计算不仅依赖当前批次数据，还需要依赖完整历史状态（需要维护历史状态）
- 方案中会明确指定模式，并给出如何维护历史状态的说明
  * 无状态模式: 无需维护历史状态
  * 半无状态模式: 在__init__函数中初始化状态变量，在特征提取函数中更新状态变量
  * 有状态模式: 在__init__函数中初始化历史特征窗口，在特征提取函数中更新历史特征窗口

## 计算流程
- 整个计算流程为事件驱动，批次数据以固定频率推送作为事件触发
- 每次事件，都会调用generate_signals函数，必须实现以下功能：
  * 特征提取: 基于当前批次数据，提取特征，并根据方案设计模式，维护历史状态
  * 信号计算: 基于当前特征和历史状态，计算信号值
  * 方向判断: 基于当前信号值，判断交易方向
- 特征提取函数，信号计算函数，方向判断函数必须分开实现，不得在同一函数中实现

# 技术框架规范
## 基础依赖
- 必须继承信号基类：`from algoUtils.baseUtil import SignalBase`
- 必须导入信号数据结构：`from algoUtils.schemaUtil import Signal`
- 必须导入得typing库: `from typing import Dict, List, Optional`
- 导入其他代码中用到的库

## 代码结构要求
- 类定义
   * 必须创建继承自`SignalBase`的类，且类名必须为`AlgoSignal`
   * 必须覆写`generate_signals`函数，该函数将被事件驱动型数据到达时多次调用

- 函数签名
   ```python
   def generate_signals(self, _data: Dict[str, List[List]]) -> Optional[List[Signal]]
   ```

## 数据结构说明
### 数据推送
- 基于在线更新框架
- 以固定频率推送批次数据
- 批次数据中包含至少一个标的，每个标的包含多条逐笔交易数据(严格按recv_ts递增)
- 信号计算频率 = 数据推送频率
- 数据推送频率不在方案中指定，由用户根据实际情况设置

### 批次数据结构示例
```python
{
    "btc_usdt|binance_future": [
        [
            recv_ts,  # float, 本地服务器收到数据的时间戳（秒，精确到小数点后6位）
            exchange_ts,    # float, 交易所发生数据的时间戳（秒，精确到小数点后6位）
            price,    # float, 成交价格
            amount,   # float, 成交数量
            direction # int, 方向(1买/-1卖)
        ],
        # ...更多逐笔交易数据
    ],
    # ...其他交易对
}
```

## 返回值结构说明
- 返回类型：`Optional[List[Signal]]`
- Signal数据结构（pydantic模型）：
```python
class Signal(BaseModel):
    batch_id: str  # str(uuid)
    symbol: str    # 'btc_usdt|binance_future'
    action: str    # 'open' or 'close'
    position: str  # 'long' or 'short'
```

## 编码规范
### 参数要求
- 信号参数平铺在__init__函数中, 并根据方案要求设置默认值
- 所有函数参数必须添加前导下划线，但实例变量名除外
- 使用蛇形命名法（snake_case）
- 示例：

  - 错误：`price_grid`（无前导下划线）
  - 错误：`_priceGrid`（使用驼峰）

- 避免使用性能较差的库（如pandas）
- 最小化循环操作
- 无需考虑工程上的优化，只需要考虑逻辑中运算性能即可
