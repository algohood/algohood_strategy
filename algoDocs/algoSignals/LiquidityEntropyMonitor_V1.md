**信号名称**：LiquidityEntropyMonitor
**版本**：V1

**信号概述**：
本信号通过测量价格区间内的成交量分布信息熵，实时评估市场流动性质量。低熵值表示成交量高度集中（流动性恶化），高熵值反映成交分散（流动性充裕）。当熵值低于历史动态阈值时，预示可能形成流动性黑洞，触发流动性危机预警。

**设计模式**：
- 模式选择：半无状态
- 选择原因：需要在实时计算中维护动态价格区间网格，采用指数衰减方式更新状态变量，平衡计算复杂度和状态维护需求

**特征提取**：
- 数据字段：
  - price（当前推送周期所有成交价格）
  - amount（对应成交量）
  - symbol（多交易对独立处理）

- 特征计算（当前批次）：
  * 当前批次特征：
    1. 动态价格网格构建：将最新价格±0.5%区间划分为20等分网格
    2. 成交量分布统计：将同批次数据按价格分配到对应网格，累计各网格成交量占比 $p_i$
    3. 香农熵计算：$H = -\sum_{i=1}^{20} p_i \ln(p_i)$

  * 状态变量更新：
    1. 定义衰减因子$\alpha=0.98$的指数加权熵值序列
    2. 更新公式：$EWMA_{new} = \alpha \times EWMA_{old} + (1-\alpha) \times H$
    3. 维护成交量累计基数：$N_{new} = \alpha \times N_{old} + \sum amount$

**信号计算**：
- 数据字段：EWMA_entropy, 当前H值
- 信号生成：
  * 归一化处理：
    计算标准化熵差：$Z = \frac{H - EWMA_{entropy}}{\sqrt{EWMA_{entropy} \times (1 - EWMA_{entropy}/N)}}$

**方向判断**：
- 数据字段：Z值
2. 交易方向生成
  * 触发条件：
    - 做空信号：当$Z < -2.5\sigma$且当前价格趋势向下（用同批次成交方向净值判断）
    - 做多信号：当$Z < -2.5\sigma$且当前价格趋势向上
    经济含义：流动性高度集中时，市场价格易发生极端波动

  * 频率控制：
    1. 使用EWMA中的条件触发比率计数器，当单位时间触发次数>3次时自动提升阈值到-3σ
    2. 价格趋势判断需满足同批次买卖方向净值比例>60%

**参数说明**：
- 阈值参数：
  - 标准差倍数：动态调节区间[-3σ,-2.5σ]
  - 触发比率重置阈值：3次/分钟
- 模式参数：
  - 网格数量：20（根据标的价格波动率可调）
  - 衰减因子：α=0.98（半衰期约34个推送周期）

**使用说明**：
- 推送频率：1秒级别（推荐1s推送周期）
- 应用场景：
  1. 主力交易时段的流动性预警
  2. 突发新闻事件后的市场结构变化监测
  3. 低流动性代币的极端波动预警

- 注意事项：
  1. 在持续单边行情中可能产生误报
  2. 需配合价格趋势过滤条件使用
  3. 不适用于稳定币交易对
