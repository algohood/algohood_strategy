**信号名称**：DynamicEMACrossover
**版本**：V1

**信号概述**：
本信号通过动态调整EMA周期参数实现智能趋势跟踪。核心思想包含两个方面：
1. 成交量加权的EMA计算：赋予大额成交数据更高权重，更真实反映市场动能
2. 自适应EMA周期：根据最新20批次的波动率自动调节EMA窗口长度，在低波动市场采用更长周期降低噪音，高波动时缩短周期提升响应速度
主要应用于15秒到3分钟级别的趋势启动阶段，预期捕捉0.8%-1.5%的价格移动。

**设计模式**：
- 模式选择：半无状态
- 选择原因：
  * 需要维护'fast_ema'、'slow_ema'两个核心状态变量
  * 各状态变量的更新仅依赖前值和新数据
  * 参数自适应机制通过增量计算实现，无需维护历史数据窗口

**特征提取**：
- 数据字段：'price'、'amount'、'direction'
- 特征计算：
  * 当前批次特征：
    + 'vwap' = Σ('price'*'amount')/Σ('amount') 
    + 'volatility' = ('max_price' - 'min_price')/'vwap' * √('total_ticks')
    + 'volume_bias' = (Σ('amount' where 'direction'=1) - Σ('amount' where 'direction'=-1))/Σ('amount')
  * 状态变量更新：
    + 'fast_ema_window' = clamp(8 + 50*(1 - sigmoid('volatility'*100)), 5, 15)
    + 'slow_ema_window' = 2.5*'fast_ema_window'
    + 'fast_ema' = coeff_fast*'vwap' + (1 - coeff_fast)*'fast_ema_prev'
      (其中 coeff_fast = 2/(1 + 'fast_ema_window'))
    + 'slow_ema' = coeff_slow*'vwap' + (1 - coeff_slow)*'slow_ema_prev'  
      (其中 coeff_slow = 2/(1 + 'slow_ema_window'))
- 复杂度分析：
  * 批次特征计算复杂度O(n)，n为当前批次数据点数
  * 状态更新为O(1)操作
  * 每个批次总体保持线性复杂度

**信号计算**：
- 计算特点：
  * 典型延迟3-5ms（取决于成交量计算）
  * 内存消耗可控制在400字节以内（2个浮点型EMA值+2个周期参数）
- 信号生成：
  * 趋势强度'ts' = ('fast_ema' - 'slow_ema') / ('vwap'*0.0001)
  * 交叉确认条件：连续3个批次的'ts'符号一致且绝对值>2σ('ts')
  * 做多信号：当条件满足且当前'volume_bias'>0.3
  * 做空信号：当条件满足且当前'volume_bias'<-0.3
  * 平仓条件：'ts'回归到±1σ区间或反向信号触发

**参数说明**：
- 阈值参数：
  * σ('ts')采用动态计算，基于最近50个批次的'ts'标准差滚动
  * 成交量偏差阈值0.3对应市场主动性参与比例>65%
  * clamp函数的窗口限制避免参数漂移
- 模式参数：
  * 基础EMA周期与波动率关系的sigmoid系数调节市场敏感度
  * 快慢窗口比例固定2.5倍保持趋势逻辑一致性
- 调优建议：
  * 在市况突变期间可临时提升σ计算窗口至100批次
  * 交易活跃时段可适当降低'volume_bias'阈值到0.25

**使用说明**：
- 推送频率：1秒（最佳实践）
- 应用场景：
  * 主要交易时段（交易所流动性充沛时）
  * 重大经济数据公布后的趋势跟随
  * 突破关键支撑/压力位后的顺势操作
- 注意事项：
  * 盘整市场需搭配波动率过滤器使用
  * 注意极端行情下EMA滞后效应
  * 建议设置最大持仓时间不超过5分钟
