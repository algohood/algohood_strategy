**因子名称**：CumulativeOrderFlow
**版本**：V5

**因子概述**：CumulativeOrderFlow 衡量订单流净量的累积效应，反映资金流动的持续性。该因子通过累加买单总量与卖单总量的净量，捕捉市场资金的净流入或净流出，帮助预测价格的潜在趋势。其理论基础基于订单流理论，累积净量能够揭示市场的潜在趋势。

**设计模式**：
- 模式选择：半无状态
- 选择原因：CumulativeOrderFlow 仅需维护一个状态变量（累积净量），并通过指数衰减控制其增长，适合半无状态模式。

**特征提取**：
- 数据字段：`amount`, `direction`

**计算流程**：
- **当前批次计算**：
  * 计算当前批次的买方总量：`buy_volume = sum(amount[direction == 1])`
  * 计算当前批次的卖方总量：`sell_volume = sum(amount[direction == -1])`
  * 计算当前批次的净量：`net_flow = buy_volume - sell_volume`

- **状态变量更新**（指数衰减法）：
  * 定义状态变量 `cumulative_net_flow`，表示累积净量。
  * 设定衰减系数 `decay_factor`（默认值：0.99）。
  * 更新规则：`cumulative_net_flow = decay_factor * cumulative_net_flow + net_flow`

- **因子输出**：
  * 因子值为 `cumulative_net_flow`，无需额外归一化。

**复杂度分析**：
- 当前批次计算：O(n)，其中 n 为批次中的数据点数。
- 状态变量更新：O(1)。
- 整体复杂度：O(n)，满足实时性要求。

**参数说明**：
- **`decay_factor`**（默认值：0.99）：
  * 数据类型：float，取值范围：(0, 1)。
  * 经济含义：控制累积净量的衰减速度。
    - `decay_factor` 越接近 1，累积净量对历史净量的记忆越长。
    - `decay_factor` 越接近 0，累积净量对当前净量的响应越快。
  * 调优建议：根据市场特性调整 `decay_factor`。例如：
    - 高频交易场景（<10s）：使用较小的 `decay_factor`（如 0.95），以更快响应市场变化。
    - 中低频交易场景（>10s）：使用较大的 `decay_factor`（如 0.99），以捕捉更长周期的趋势。

**使用说明**：
- **推送频率**：1s~1min。
- **应用场景**：
  * 高频交易：捕捉短期资金流动趋势。
  * 中低频交易：监测市场资金流动的长期趋势。
- **注意事项**：
  * `decay_factor` 的选择对因子性能有重要影响，需根据具体场景进行调优。
  * 在极端市场条件下（如极端波动或流动性不足），累积净量可能出现偏差，建议结合其他指标使用。
